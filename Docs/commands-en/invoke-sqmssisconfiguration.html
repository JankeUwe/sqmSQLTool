Invoke-sqmSSISConfiguration
ConfigurationsqmSQLTool v1.8.2+ · SSIS Setup⚠ Requires SQL 2012+  ✓ Full Catalog Config
Configures SQL Server Integration Services (SSIS) including: catalog creation, folder setup, project/environment permissions, retention policies, server-level settings, and validation. Supports both SSIS Catalog (SQL 2012+) and legacy SSIS Package Store configurations.

Parameters

ParameterTypeRequiredDefaultNotes
-SqlInstancestringRequired—Target SQL Server instance (SSIS host).
-SqlCredentialPSCredentialOptional—Credentials for SSIS connection.
-ActionstringRequired—CreateCatalog | CreateFolder | ConfigurePermissions | ConfigurePolicy
-FolderNamestringOptional—Folder name for Action=CreateFolder.
-FolderDescriptionstringOptional—Folder description.
-EnvironmentNamestringOptional—Environment name for ConfigurePermissions action.
-RetentionDaysintOptional365Operation log retention in days (Action=ConfigurePolicy).
-EnableExceptionswitchSwitch$falseRe-throw exceptions immediately.
-WhatIf / -ConfirmswitchOptional—ShouldProcess (ConfirmImpact: Medium) guards configuration.

Execution Flow

Examples

Create SSIS Catalog
Invoke-sqmSSISConfiguration -SqlInstance "SQL01" -Action "CreateCatalog"
Create folder and set retention policy
Invoke-sqmSSISConfiguration -SqlInstance "SQL01" -Action "CreateFolder" -FolderName "Production" -FolderDescription "Production SSIS Projects"
Invoke-sqmSSISConfiguration -SqlInstance "SQL01" -Action "ConfigurePolicy" -RetentionDays 180
